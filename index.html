Theory Code

  • An Awkward State Machine (ASM) is composed of two main types of objects:

    • Cycles
    • A Branch

    Each 'cycle' is composed of a list of nodes connected into a circle (think beads on a necklace). While a branch is a cycle that hasn't been connected into a circle yet (someone cut your bead necklacke, giving you a line segment). Currently, both cycles and branches will be represented as horizontal bars whose nodes consist of squares and circles.
    Both cycles and branches keep track of a position node. When the ASM moves from one state to the next, each cycle's position is moved one node to the right (if you are at the rightmost position, the position gets set to the leftmost node). After moving every cycle in the ASM, the ASM checks whether or not it is 'active'. An ASM is considered active if one of its cycle's position is on one of its activator nodes; thus an ASM is 'inactive' only if all of the cycles' position's are on non-activator nodes.

    If the ASM is 'inactive' after moving every cycle, then the following will occur:
    • The current branch will be copied
    • The original branch will be closed into a cycle. It's position will be set to 0 (its leftmost nodes) and that will activate the ASM.
    • The copied branch becomes the only branch.
    • A new node is added to the end of the branch (the new node becomes the rightmost node). The branch's position is set on the new node.

    If the ASM was 'active' after moving every cycle, then:
    • A new node is added to the end of the branch (the new node becomes the rightmost node). The branch's position is set on the new node.

    Therefore, regardless of whether the ASM was inactive/active, a new node is added to the ASM every iteration.

    The initial state of an ASM consists of an initial cycle and a branch. When definining an ASM, you must first define the number of activator nodes it will contain, and the number of non-activator nodes the initial cycle will have (see below). The first (leftmost) nodes of an ASM's cycle are always the activator nodes, which are then followed by the non-activator nodes. Finally, the initial branch is one node longer than the initial cycle (#activator + #non-activator + 1).

  • Number of Activators:

    The number of activator nodes each cycle will contain. A cycle is 'active' it's current position is on one of the activator nodes. The activator nodes are represented as squares. The cycle's position is on a square if the square is empty (rather than filled in with blue).


  • Number of Non-Activators:

    The number of non-activator nodes the first cycle will contain (giving it a length of [#activator + #non-activator]). A cycle is 'inactive' it's current position is on one of the activator nodes. The non-activator nodes are represented as circles. The cycle's position is on a circle if the circle is empty (rather than filled in with blue).
